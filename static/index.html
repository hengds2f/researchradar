Section Classifier

SciBERT-style zero-shot classification assigns scientific section labels with confidence scores.

How this works

This tool reads every paragraph in a paper and uses machine learning to classify it:

  • Each paragraph is labelled with a section type (e.g., Introduction, Methods, Results)
  • A confidence score (0–100%) shows how certain the model is about each label
  • Two labels are shown: the original heading found in the paper, and the AI's predicted label
  • This helps you quickly understand the structure of an unfamiliar paper

    Five specialised AI agents work together in sequence to produce a high-quality, well-evidenced answer:

    🔬 Critic — Reviews the draft for weak claims, unsupported statements, or logical gaps, and scores its quality
    ✍️ Writer — Rewrites the final response incorporating the critic's feedback for clarity and accuracy

    Provenance Ledger

    An automatic, tamper-proof record of everything that happens to your research papers.

    What is provenance and why does it matter?

    Provenance simply means proof of origin and history — the same idea as a certificate of authenticity for a piece of art, or a chain of custody for legal evidence.

    In research, provenance answers questions like: "Was this paper really uploaded unchanged?", "When exactly did the AI generate this summary?", "Can I prove I had this finding before a competitor published theirs?"

    ResearchRadar builds this proof automatically by:

    • Fingerprinting every action — When you upload a paper or the AI generates an output, the system computes a unique "fingerprint" (called a hash) of that content. Even a single changed character produces a completely different fingerprint, making silent tampering detectable.
    • Chaining records together — Each new record includes the fingerprint of the previous one, like links in a chain. If any record is altered retroactively, every subsequent link breaks — making the tampering obvious.
    • Timestamping events — Every record is stamped with the exact date and time it was created, giving you a verifiable timeline of your research activity.

    Think of it like a bank statement: you cannot silently edit a past transaction without the statement becoming invalid. This ledger works the same way for your research.

    ⬇ Export Chain JSON — what does this do?

    Downloads the entire provenance chain for the selected paper as a JSON file you can save to your computer.

    What's inside: Every event (upload, AI outputs, timestamps, fingerprints) for that paper, plus a chain validity report.

    Why is this useful?

    • You can share this file with a journal editor, grant reviewer, or institutional repository as machine-readable proof that your paper and its analysis were produced on specific dates
    • A technical collaborator can independently re-run the hash checks without needing to visit this site
    • You can archive it alongside your paper as part of your reproducibility package — funders like the NIH and UKRI increasingly require this
    • Audit scripts or CI/CD pipelines can load this file to automatically verify dataset integrity before a publication goes live
    📜 Download Proof Doc — what does this do?

    Downloads a structured proof document — a single JSON file designed to be submitted as evidence of authorship or date-of-discovery.

    What's inside: A clean summary of the paper's title and filename, the exact timestamp of its first upload, the content fingerprint at upload time, any blockchain transaction reference, any IPFS storage reference, and a chain validity verdict.

    Why is this useful?

    • Intellectual property disputes — If two researchers publish similar findings and the question of "who had it first" arises, this document is timestamped proof that your version existed on a specific date
    • Grant reporting — Many funders ask for evidence of when data was collected or analysed. This document provides that in a verifiable format
    • Journal submission — Some journals accept a provenance document alongside a submission to demonstrate research integrity
    • Institutional compliance — Universities with open research or data governance policies can accept this file as part of the audit trail

    Unlike the full chain export, this document is intentionally concise — one page of key facts — so it can be attached to an email or uploaded to a submission portal without overwhelming the recipient.

    Provenance History

    A live timeline of every recorded event for your paper — uploads, AI outputs, and agent runs.

    Reading the timeline — what do all these numbers mean?

    Each card in the timeline represents one recorded event. Here is what the technical terms mean in plain English:

    • 📄 Upload event — Created the moment you uploaded a paper. Proves the file existed and was unchanged at that exact time.
    • 📝 Summary event — Created when the AI generated a synthesis or analysis. Records what the AI produced so the output can never be quietly altered later.
    • 🤖 Agent Output event — Created after a multi-agent workflow completes. Includes the quality score and confidence level from the Critic agent.

    The four identifiers on each card:

    • Content Hash — A 64-character code that is a mathematical fingerprint of the actual content (the paper text, or the AI's output). If the content changes even slightly, this code changes completely. You can independently verify it by running sha256sum on the file yourself and comparing.
    • Record Hash — A fingerprint that covers both this record and the previous record's hash. This is what creates the unbreakable chain — it's impossible to change an old record without breaking every hash that comes after it.
    • Tx Hash (when blockchain is enabled) — A reference number for the Ethereum transaction that anchored this fingerprint on the public blockchain. Clicking the ↗ link opens the transaction on a blockchain explorer like Etherscan, where anyone in the world can independently verify it — no login required, and the record exists even if this website shuts down.
    • IPFS CID (when IPFS is enabled) — A permanent address for the content stored on the InterPlanetary File System, a global decentralised storage network. Clicking the ↗ link opens the file via a public gateway. Like the blockchain anchor, this is accessible by anyone and survives independently of this site.

    Click the ⧉ copy button next to any hash to copy the full value to your clipboard for use in external tools or reports.

    🔍 Verify a Hash

    Check whether a specific content fingerprint exists in the provenance chain — without needing to load the full timeline.

    What is this for and how do I use it?

    What it does: You paste a SHA-256 hash (a 64-character string of letters and numbers) and the system searches every provenance record to find a match. If found, it tells you which paper it belongs to, what type of event it was, and when it was recorded.

    Where does a hash come from?

    • Copy a Content Hash from any provenance card above and paste it here to confirm it is genuine
    • Run sha256sum yourpaper.pdf in a terminal on your own computer and paste the result — if it matches a record here, you have independent proof that the file was registered unchanged
    • A journal editor, reviewer, or collaborator can use this tool to verify a hash you sent them without needing an account or log-in

    Practical examples:

    • A co-author questions whether the version of the paper they received is the same as what was uploaded — paste the hash of their copy here to check
    • A reviewer wants to confirm that the AI-generated analysis they are reading was produced at the time you claim — paste the content hash from your proof document
    • An automated pipeline (e.g. a GitHub Action or a submission portal) can call the /api/provenance/verify-hash API endpoint directly to validate integrity as part of a workflow

    The result will tell you: ✓ found (with details), or ✗ not found (meaning the content was never registered, or was registered in a different session).
